iOS – NSUserDefaults

NSUserDefaults适用于存储轻量级的本地数据，例如用户的登录名，密码，URL之类的。

使用NSUserDefaults存储基本数据类型

NSUserDefaults支持的数据类型有floats, doubles, integers, Booleans, and URLs 也支持NSData, NSString, NSNumber, NSDate, NSArray, or NSDictionary。

在Apple的官方文档中有一段这样的话

Values returned from NSUserDefaults are immutable, even if you set a mutable object as the value. For example, if you set a mutable string as the value for “MyStringDefault”, the string you later retrieve using stringForKey: will be immutable.

说明NSUserDefaults的存储对象是不可变的，如果不遵守这个规则程序就出现bug，如果要存储一个可变的数组，必须把它转变成不可变的数组再存入NSUserDefaults

使用NSUserDefaults存储其他对象

有一个场景，基本上APP都会有个人信息一栏，其中的个人头像其实也可以存储在NSUserDefaults，从服务器获取到的头像载入到移动端，可以将UIImage转化成NSData，存到本地。

总结

使用NSUserDefaults存储数据是最直接、最简单的方式
